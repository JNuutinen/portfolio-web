Trainee @ Planmeca

May 2018 March 2019
Robot Framework

This is where I got my first introduction to test automation and QA, as a trainee at Planmeca 's Dental Unit R&D team.

I set up automated tests for Planmeca 's Romexis, which is a Java based desktop application. I did automated end to end user acceptance tests for the graphical UI. Another part of the testing was related to systems integration, where the tests would trigger events from the UI, that would in turn start processes in external systems. An example of this was starting a dental unit 's software update from the desktop app.

I also worked on embedded systems automation for dental units. The software update process for dental units was laborious to test manually: it took a long time, and there were a lot of different version increments to test. Automating tests for this flow was interesting, because it required managing multiple different systems at the same time. Running the robot tests on a computer, connecting to the embedded system of the dental unit, managing the software files on a USB stick connected to the unit, and giving instructions programmatically to the touch UI of the dental unit to simulate a user.
